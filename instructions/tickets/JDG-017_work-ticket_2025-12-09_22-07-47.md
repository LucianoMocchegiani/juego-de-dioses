## Descripción

### Problema/Requerimiento

Existen varios archivos de documentación temporal en la raíz del proyecto que fueron creados durante el desarrollo inicial y ya no son necesarios. Estos archivos generan confusión y hacen difícil encontrar la documentación relevante. Es necesario limpiar estos archivos y asegurar que el README principal esté actualizado y contenga toda la información esencial.

### Comportamiento Actual

Existen los siguientes archivos de documentación temporal en la raíz del proyecto:
- `GUIA-CREAR-MODELOS-PARTES-CRITICAS.md`
- `INICIO-RAPIDO.md`
- `INSTRUCCIONES-DESCARGAR-MODELO.md`
- `INSTRUCCIONES-PRUEBA-JDG-010.md`
- `PROMPTS-IA-MODELOS-VOXEL.md`

Además, existen scripts de utilidad temporal en `backend/src/database/` que ya no se utilizan:
- `download_test_model.py` - Script para descargar modelos de prueba (no se usa)
- `cleanup_duplicate_characters.py` - Script para limpiar personajes duplicados (no se usa)
- `update_human_scale.py` - Script para actualizar escala de modelo (no se usa)

También se identifican problemas de formato:
- Los READMEs del proyecto contienen emojis, lo que no mantiene un tono profesional
- La sección "Próximos Pasos" está en el README principal en lugar de un archivo dedicado
- No hay una regla explícita que prohíba el uso de emojis en READMEs

Estos archivos contienen información que ya debería estar en el README principal o en documentación más estructurada dentro de `instructions/`.

### Comportamiento Esperado

- Los archivos de documentación temporal son eliminados
- Los scripts de base de datos no utilizados son eliminados
- Todos los READMEs del proyecto están sin emojis (tono profesional)
- La regla `readme-rule.mdc` incluye prohibición explícita de emojis
- El README principal (`README.md`) contiene toda la información esencial de inicio rápido y referencia, sin la sección "Próximos Pasos"
- Se crea `roadmap.md` en la raíz con los próximos pasos del proyecto
- La documentación está organizada de forma clara y accesible
- No hay archivos obsoletos o redundantes en la raíz del proyecto ni scripts no utilizados en `backend/src/database/`
